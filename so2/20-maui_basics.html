MAUI - Multi-platform App UI

Waarom MAUI?

Tot nu toe hebben we vooral gewerkt met console-applicaties, met een eerste uitstap naar MonoGame. MonoGame liet ons zien dat softwareontwerp niet beperkt blijft tot theorie of tekstoutput, maar ook toegepast kan worden in echte, interactieve toepassingen.

Die lijn trekken we nu verder met .NET MAUI.

Hoe meer je ontwerpt en programmeert, hoe beter je wordt en hoe meer je ontdekt dat programmeren ook plezier kan brengen. De keuze voor MAUI en MonoGame past precies binnen dat idee: we willen programmeren boeiender en leuker maken dan enkel werken in de console, zonder de focus op goed ontwerp te verliezen.

Wat is MAUI (Multi-platform App UI)

.NET MAUI (Multi-platform App UI) is het framework van Microsoft waarmee je met één codebasis applicaties kan maken voor:

  • Android

  • iOS

  • Windows

  • macOS

  • Tizen (door Samsung)

Met MAUI schrijf je dus één project dat op meerdere toestellen kan draaien. Dezelfde logica, dezelfde schermen, dezelfde taal.

De MAUI workload

MAUI-applicaties kunnen ontwikkeld worden in Visual Studio .NET, Visual Studio Code of JetBrains Rider. In deze lessen gebruiken we Visual Studio .NET, maar het staat je vrij om een ander IDE te gebruiken als je dat verkiest.

Om een MAUI-project te kunnen aanmaken, moet Visual Studio weten wat MAUI is. Dat gebeurt via een workload: een optionele uitbreiding die ondersteuning toevoegt voor mobiele en multi-platform ontwikkeling.

Installeren van de MAUI workload

  1. Open Visual Studio Installer. Je krijgt een lijst met alle gevonden Visual Studio-installaties.

  2. Klik op Modify bij de versie van Visual Studio waarin je MAUI wilt installeren.

  3. In de tab Workloads, onder Desktop & Mobile, vind je de workload ".NET Multi-platform App UI development". Vink deze aan.

  4. Klik rechtsonder op Modify om de installatie te starten.

Na installatie beschik je over de nodige projecttemplates voor MAUI.

Een eerste .NET MAUI App

Zodra de workload geïnstalleerd is, kan je een nieuw project aanmaken met de template ".NET MAUI App".

In MAUI maak je schermen op basis van twee gekoppelde bestanden:

  • een XAML-bestand waarin je beschrijft hoe het scherm eruitziet

  • een code-behind-bestand (C#) waarin de logica van dat scherm zit

Je zal nu in je solution explorer iets zien zoals het onderstaande

Platforms

Deze map bevat de platformspecifieke code voor Android, iOS, Windows en eventueel macOS. Wanneer je iets wil doen dat enkel op één platform bestaat (zoals een Android-specifieke permissie of een Windows-specifieke instelling), dan komt dat hier terecht. De meeste code die je schrijft zal platformonafhankelijk zijn, maar MAUI laat toe om via deze mappen in te grijpen als het echt nodig is.

Resources

Hier staan alle resources van de app, zoals afbeeldingen, fonts, stijlen, kleuren en ruwe bestanden. MAUI verwerkt deze bij het bouwen en maakt ze beschikbaar op elk platform. Een afbeelding die je in deze map plaatst, kan je dus rechtstreeks gebruiken in XAML zonder dat je ze per platform moet kopiëren.

App.xaml

Je kan hier onder andere globale stijlen, kleuren en thema’s definiëren die gelden voor alle pagina’s van de app. Het hoort samen met de App.xaml.cs code-behind, die de startlogica van de applicatie bevat (zoals welke pagina eerst getoond wordt).

AppShell.xaml

AppShell bepaalt de navigatiestructuur van de app. Hierin beschrijf je welke pagina’s er als root bestaan en hoe je ertussen navigeert (bijvoorbeeld via een menu of tabbladen). MAUI gebruikt het concept van Shell om navigatie eenvoudiger en overzichtelijker te maken dan vroeger in Xamarin.

MainPage.xaml

Dit is het eerste scherm dat getoond wordt wanneer de app start. Dit is vaak de plaats die je aanpast om je eerst scherm te hebben.

MainProgram.cs

MauiProgram is het instappunt van de applicatie, een beetje zoals bij een console applicatie. Hier wordt de app geconfigureerd en opgestart. Je ziet er meestal een CreateMauiApp()-methode die de applicatie bouwt via een MauiAppBuilder.

De XAML

<?xml version="1.0" encoding="utf-8" ?>
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
             xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
             x:Class="MauiApp.MainPage">

    <ScrollView>
        <VerticalStackLayout
            Padding="30,0"
            Spacing="25">
            <Image
                Source="dotnet_bot.png"
                HeightRequest="185"
                Aspect="AspectFit"
                SemanticProperties.Description="dot net bot in a hovercraft number nine" />

            <Label
                Text="Hello, World!"
                Style="{StaticResource Headline}"
                SemanticProperties.HeadingLevel="Level1" />

            <Label
                Text="Welcome to &#10;.NET Multi-platform App UI"
                Style="{StaticResource SubHeadline}"
                SemanticProperties.HeadingLevel="Level2"
                SemanticProperties.Description="Welcome to dot net Multi platform App U I" />

            <Button
                x:Name="CounterBtn"
                Text="Click me"
                SemanticProperties.Hint="Counts the number of times you click"
                Clicked="OnCounterClicked"
                HorizontalOptions="Fill" />
        </VerticalStackLayout>
    </ScrollView>

</ContentPage>

De code-behind

public partial class MainPage : ContentPage
{
    int count = 0;

    public MainPage()
    {
        InitializeComponent();
    }

    private void OnCounterClicked(object? sender, EventArgs e)
    {
        count++;

        if (count == 1)
            CounterBtn.Text = $"Clicked {count} time";
        else
            CounterBtn.Text = $"Clicked {count} times";

        SemanticScreenReader.Announce(CounterBtn.Text);
    }
}

Wanneer je deze app uitvoert, krijg je een eenvoudige gebruikersinterface met een image, een knop en een paar labels. Klik je op de knop, dan wordt de tekst van de knop aangepast.

Layouts en controls

MAUI biedt verschillende containers om views in te plaatsen. Veel gebruikte zijn:

  • VerticalStackLayout

  • HorizontalStackLayout

  • Grid

Een overzicht van alle beschikbare user interface elementen vind je hier: https://learn.microsoft.com/en-us/dotnet/maui/user-interface/controls/?view=net-maui-10.0

Oefening

Maak een kleine app die een willekeurige grap toont uit een hardcoded lijst. Het scherm moet:

  • Een knop bevatten die een willekeurige grap uit de lijst toont in de label die boven de knop staat

  • Een knop om een nieuwe grap aan de lijst toe te voegen, dewelke getypt is in het invoerveld links naast de knop

Denk in objectgeoriënteerde termen en het drielagen model, de lijst met grappen hoort niet rechtstreeks in de code-behind als List<string>
